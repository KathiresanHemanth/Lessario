Build native-quality mobile apps for iOS & Android. Flutter and React Native let you ship to both platforms from one codebase.

## Core Concepts

1. **Dart (Flutter)** or **JavaScript/TypeScript (React Native)**
2. **UI Components** — Widgets/Components, layouts, navigation
3. **State Management** — Provider/Riverpod (Flutter), Redux/Zustand (RN)
4. **API Integration** — REST, GraphQL, WebSocket
5. **Local Storage** — SQLite, Hive, SharedPreferences, AsyncStorage
6. **Authentication** — Firebase Auth, OAuth, biometrics
7. **Push Notifications** — FCM, APNs, local notifications
8. **Camera & Sensors** — Camera, GPS, accelerometer, Bluetooth
9. **Animations** — Implicit/explicit animations, Lottie, Rive
10. **App Store Deployment** — Play Store, App Store, code signing
11. **Performance** — Profiling, lazy loading, image optimization

## Flutter vs React Native (2026)

| Aspect | Flutter | React Native |
|--------|---------|-------------|
| **Language** | Dart | JavaScript/TypeScript |
| **Performance** | Native-compiled | Bridge-based (improving) |
| **UI** | Custom rendering (Skia) | Native components |
| **Hot Reload** | ✅ Excellent | ✅ Good |
| **Adoption** | Growing fast (Google) | Established (Meta) |
| **Web/Desktop** | ✅ Built-in | Partial |
| **Best For** | Custom UI, multi-platform | JS teams, existing web devs |

## Tips
- **Pick Flutter OR React Native** — don't learn both initially
- **Flutter if new to mobile** — cleaner architecture, better DX
- **React Native if you know JS** — leverage existing skills
- **Build real apps and publish** — Play Store listing > any cert
- **Firebase** is your best friend for backend-as-a-service
