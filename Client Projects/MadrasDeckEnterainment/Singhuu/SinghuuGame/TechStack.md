# Technology Stack & Output Expectations

> *Technical framework and deliverables for "Nama Thala Singhuu: Ekam Run"*

---

## 1. Technology Stack

Because you are using me (an AI coding assistant) for the rapid prototyping phase, we need a tech stack that is built for instant visibility, zero-compilation wait times, and immediate playability directly on your PC. 

### A. Prototyping Engine (Phase 1)
**Primary Option:** **Web Technologies (HTML5 Canvas / Three.js / JavaScript)**.
*   **Why:** I can write a functional, playable prototype of the core logic right now. You can open it instantly in any web browser without downloading SDKs, setting up Unity, or compiling APKs. It captures the exact game logic (bullet time, obstacle parsing, movement) in an agile way.

### B. Production Engine (Phase 2 & Scaling)
**Primary Option:** **Unity 3D** or **Lessario Engine**.
*   **Why:** Once the Web Prototype proves the game is fun and the logic works, your development team will rewrite the final assets and logic into Unity or your proprietary engine to build the optimized Android/iOS mobile versions.

### C. Pipeline & Art
*   **AI Integration:** **Google AI Studio** — Used to rapidly sequence the trivia questions and dialogue.
*   **Art & UI:** Maya/Blender for 3D models, Figma for final mobile interface testing.

---

## 2. Deliverables & What to Expect

Here is the structured roadmap of outputs for this project:

### Phase 1: Interactive Web Prototype (AI Assisted)
*   **Output:** Local HTML/JS files playable in your Desktop browser (locked to mobile aspect ratio).
*   **Expectation:** A fast, functional playground built entirely by me to prove the core loop. It will feature basic shapes dodging obstacles, and specifically demonstrate the slow-motion triva system.

### Phase 2: Vertical Slice (Alpha)
*   **Output:** High-fidelity Android build (`.aab`).
*   **Expectation:** One fully realized biome ("The Dense Forest"), the completed main character (Singu) with final animations, and integrated UI. This is your "investor/stakeholder pitch" build.

### Phase 3: Android Beta (Soft Launch)
*   **Output:** Google Play Store Beta Release.
*   **Expectation:** All 3 starter characters, 2-3 procedurally generated biomes, and a complete question bank. Monetization SDKs (Ad integration) are implemented. We will gather analytics on child engagement and fix bugs.

### Phase 4: Full Android Release & LiveOps
*   **Output:** Public Version 1.0 on Google Play.
*   **Expectation:** Continuous connection to the Chutti TV show — new episodes result in updates to the in-game trivia and events via a LiveOps pipeline.

### Phase 5: iOS Port
*   **Output:** App Store Release (`.ipa`).
*   **Expectation:** Once Android retention is proven, the Unity/Lessario project is optimized for Apple devices, utilizing Metal rendering, and submitted to the App Store.
